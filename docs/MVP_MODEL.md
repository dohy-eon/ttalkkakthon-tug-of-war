# 줄?다리기? MVP 모델 (기능명세 반영)

## 1) 목표
- 로그인 없이 30초 내에 바로 플레이 가능한 자이로 기반 웹 줄?다리기? 경험 제공
- PC(방장 화면)와 모바일(조작기) 간 실시간 동기화 검증
- 2인/팀전 중심의 멀티플레이 줄다리기 경험 제공

## 2) MVP 범위 정의

### 포함 (Must Have)
- 공통
  - 닉네임 입력(2~10자, 공백 불가, 금칙어 필터)
  - 센서 권한 요청/재시도/미지원 안내
  - 센서 캘리브레이션(기준값 저장, 재설정)
  - 결과 화면
- 2인 모드
  - PC 방 생성(방 코드 + QR 표시)
  - 모바일 방 코드/QR 입장
  - 실시간 줄 이동(중앙 마커, 승리 라인)
  - 30초 제한 + 선도달 즉시 승리
  - 결과 요약(승패, 최종 위치, 개인 기여도)
 - 팀전 모드
  - PC 방 생성 후 팀 인원 밸런스 체크
  - 모바일 팀 선택/변경
  - 팀별 합산 힘 기반 실시간 줄 이동
  - 결과 요약(승패, 팀/개인 기여도)

### 제외 (Won't Have for MVP)
- SYNC/PERFECT SYNC 보너스
- 명예/불명예 전당(사진 업로드/동의 플로우)
- 고도화 안티치트(복합 이상패턴 ML 수준)
- 고급 운영정책(동률 상세 규칙 다변화, 밴/제재 기능)

## 3) 모드별 성공 기준(Release Gate)

### 2인/팀전 모드
- 방 생성 후 모바일 입장, 시작, 종료까지 끊김 없는 플로우
- 줄 위치가 실시간으로 갱신되고 양측 입력 우세가 체감적으로 반영
- 30초 타임아웃/선도달 승리 규칙이 항상 일관되게 동작

## 4) 핵심 상태 모델

### 방 상태
- `대기중` -> `참가중` -> `준비완료` -> `게임중` -> `결과표시` -> `종료`

### 플레이어 상태
- `입장완료` -> `권한허용완료` -> `캘리브레이션완료` -> `준비완료` -> `플레이중` -> `결과확인`

### 게임 상태
- `카운트다운` -> `일반진행` -> `피버타임` -> (`승리확정` | `시간종료`) -> `결과화면`

## 5) 센서 처리 MVP 규칙

### 입력 수집
- 주기: 20~50ms 단위 샘플링
- 입력: 좌우 축 중심의 가속/기울기 값
- 노이즈 필터: 이동평균 또는 지수감쇠(EMA) 1종 필수

### 유효 판정
- 최소 힘 임계치 이상
- 캘리브레이션 기준 대비 수평 오차 허용 범위 내
- 비정상 급등치/너무 빠른 연타는 감쇠 또는 무효

### 판정 등급(간소화)
- `PERFECT`, `GOOD`, `WEAK`, `INVALID`
- MVP에서는 서버 계산식 단순화(선형 가중치) 허용

## 6) 점수/힘 계산 모델 (MVP 권장안)

### 2인/팀전 힘 반영
- 플레이어별 유효힘 계산 후 팀 평균(또는 합) 반영
- `delta = (teamA_force - teamB_force) * K`
- `position = clamp(position + delta, -100, 100)`
- `position`이 승리 라인 도달 시 즉시 종료

## 7) 실시간 동기화 MVP 요구
- 서버 authoritative 방식으로 승패/줄 위치 확정
- 브로드캐스트 주기: 20Hz 권장(최소 10Hz)
- 클라이언트는 시각 보간만 수행, 승패 판정은 서버 단일화

## 8) 데이터 모델 (MVP 최소 스키마)

### Player
- `playerId` (internal UUID)
- `nickname`
- `roomId`
- `state`
- `calibration` (baseline tilt)

### Room (2인)
- `roomId`
- `mode = duel`
- `status`
- `players[]`
- `position`
- `startedAt`, `endedAt`

### MatchResult
- `matchId`
- `mode` (`duel` | `team`)
- `winner` (duel only)
- `durationSec`
- `stats` (combo, accuracy, contribution, score)
- `createdAt`

## 9) API/소켓 이벤트 MVP 기준

### 공통
- `nickname:validate`
- `sensor:permission`
- `sensor:calibrate`

### 2인/팀전 모드
- `room:create`
- `room:join`
- `room:ready`
- `game:start`
- `game:input`
- `game:state`
- `game:end`

## 10) 예외 처리 MVP 우선순위
- 네트워크 끊김: 플레이어 비활성 + 재연결 유예시간(예: 5초)
- 센서 미지원/권한 거부: 재시도 + 가이드 문구 제공
- 인원 미충족: 시작 버튼 비활성
- 경기 중 이탈: 2인 모드는 상대 승 처리(정책 고정)

## 11) 단계별 구현 로드맵

### Phase 1 (즉시 개발)
- 공통 입력(닉네임/권한/캘리브레이션)
- 2인/팀전 기본 루프(방, 입장, 실시간 줄 이동, 승패)

### Phase 2 (MVP 안정화)
- 입력 판정 튜닝(노이즈/비정상 처리 강화)
- UI 피드백 고도화(카운트다운, 효과, 결과 통계)

### Phase 3 (확장)
- 팀전 + SYNC 시스템
- 명예/불명예 전당(사진 업로드/동의)

## 12) 현재 코드 대비 갭 요약
- 이미 구현됨
  - 실시간 방 생성/입장
  - 기본 줄 이동 및 승패 판정
  - 모바일 센서 입력 전송
- 추가 필요(MVP 기준)
  - 닉네임 검증 규칙(필수/길이/금칙어)
  - 센서 권한 실패/미지원 UX 보강
  - 캘리브레이션 화면/기준값 반영
  - 30초 타이머 + 피버타임 + 결과 통계
  - 팀전 UX 안정화 및 팀 밸런싱 정책
